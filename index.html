Scalable fault tolerant storage for PostgreSQL

Zenith substitutes PostgreSQL storage layer and redistributes data across a highly-available cluster of nodes

Subscribe to updates:

Features

PostgreSQL compatibility

Full app and extension compatibility with PostgreSQL. Compute node on top of the Zenith cluster is a minimally modified PostgreSQL instance.

Built for cloud

Scale volume size and performance by dynamically adding nodes in the storage cluster.

Developer tools

Zenith CLI tools allow users to easily export/import/compare and deploy to cloud their data.

Fault tolerance

Zenith uses consensus-based replication to achieve availability in presence of failures.

Instant recovery

Zenith is able to start a failed PostgreSQL instance without a recovery stage in a matter of seconds. PostgreSQL node running on top of Zenith is considered stateless and does not requires persistent storage.

Snapshots

Zenith storage allows taking instantaneous Copy-On-Write snapshots of the database. Snapshots in Zenith initially consumes no storage space and will grow proportionally to the amount of changed data from snapshot.
